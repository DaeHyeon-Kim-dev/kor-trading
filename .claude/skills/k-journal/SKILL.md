# 데일리 저널 (k-journal)

매일 예측을 기록(record)하고, 다음 거래일에 실제가로 검증(review)하는 피드백 루프.

**기록 (장 마감 후/하루 시작 시):**
```bash
cd "${CLAUDE_SKILL_DIR}/../../.." && export PATH="$HOME/.local/bin:$PATH" && uv run python "${CLAUDE_SKILL_DIR}/scripts/run.py" record
```

**검증 (다음 거래일):**
```bash
cd "${CLAUDE_SKILL_DIR}/../../.." && export PATH="$HOME/.local/bin:$PATH" && uv run python "${CLAUDE_SKILL_DIR}/scripts/run.py" review
```

스크리너를 돌리므로 1~2분 걸릴 수 있다. 오래 걸리면 백그라운드로 실행.

## 출력 활용 — 중요
이 스킬은 **사실(숫자)** 만 만든다. 너는 그 위에 **분석과 학습**을 더해 진짜 데일리 기록을 완성한다:

1. **record 결과**를 보여준 뒤, 그날 시장·셋업에 대한 짧은 코멘트를 단다.
2. **review 결과**가 핵심이다. 표(예측 vs 실제)를 보여준 뒤 반드시:
   - **맞은 예측**: 왜 통했나(어떤 셋업·레짐에서). 
   - **틀린 예측**: 왜 빗나갔나(레짐 불일치? 셋업 약함? 갭? 뉴스?). 가능하면 `/k-disclosures`로 원인 공시 확인.
   - **패턴/학습**: 누적해서 "어떤 셋업이 어떤 장에서 잘/못 맞는지" 한 줄 인사이트. 이게 개선의 씨앗.
3. **방향 적중률(1일~며칠)은 노이즈가 크다**는 점을 명시 — 셋업의 진짜 성과는 목표/손절 도달(상태)로 본다.
4. 누적 검증이 쌓이면 **백테스트 기대값(돌파 +0.53R 등) vs 실제 forward 성과**를 대조해, 괴리가 크면 셋업 파라미터 재검토를 제안한다.

## 자동화
매 거래일 자동 기록·검증을 원하면 `/schedule` 또는 `/loop`로 이 스킬을 거래일 루틴에 건다(예: 평일 16:00 record, 다음날 09:30 review).
