# 셋업 백테스트 (k-backtest)

과거 일봉으로 셋업 진입→손절/목표 청산을 시뮬레이션해 기대값을 낸다.

```bash
cd "${CLAUDE_SKILL_DIR}/../../.." && export PATH="$HOME/.local/bin:$PATH" && uv run python "${CLAUDE_SKILL_DIR}/scripts/run.py" $ARGUMENTS
```

수 분 걸릴 수 있다(종목당 과거 전체를 워크포워드). 오래 걸리면 백그라운드로 실행한다.

## 출력 활용
- 셋업별 표(거래수·승률·기대값R·손익비·MDD)를 그대로 보여준다.
- **기대값(R)이 핵심**: >0이면 양의 기대값(유효), <0이면 손실 셋업이다.
- 승률이 낮아도 손익비가 높으면 기대값은 +일 수 있음을 짚어준다(반대도 마찬가지).
- 표본(거래수)이 적으면(예: <30) "통계적으로 신뢰 부족"이라고 솔직히 경고한다.
- 수급주도 셋업은 과거 수급 데이터 부재로 백테스트에서 빠진다는 점을 안내한다.
